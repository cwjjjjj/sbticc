# SBTI 变现方案设计

## 背景

- 短期爆发型产品，约 1000 用户/2小时，日均 ~12000 PV
- 中国和海外用户各约 50%
- 无现有支付/广告平台账号
- 目标：1-2 天内上线，激进变现
- 网站域名：sbti.jiligulu.xyz

## 方案概述：广告 + 轻付费组合

广告保底覆盖所有用户，付费解锁详细报告作为增量收入，分享图水印驱动裂变拉新。

---

## 一、广告系统

### 广告平台

使用 **Monetag**（或 PropellerAds）统一覆盖国内外流量。

选择理由：
- 个人可注册，无需企业资质
- 即时审批，注册几小时内可跑广告（AdSense 需 1-2 周）
- 自动按 GEO 优化，无需区分国内外用户
- 支持 PayPal / USDT 提现

### 广告位

| 位置 | 格式 | 触发时机 |
|------|------|----------|
| 结果页插屏 | Interstitial 全屏广告 | 提交答案后，展示 5 秒倒计时，结束后显示结果 |
| 结果页底部 | Banner 300x250 或 728x90 | 结果卡片下方常驻 |
| 排行榜信息流 | 原生广告 | 排行列表每 10 条插入一条 |

### 实现方式

- `<script>` 标签引入 Monetag SDK
- 插屏广告使用 Monetag Interstitial API
- Banner 使用标准广告容器 `<div>`
- 付费用户隐藏所有广告位

## 二、付费解锁系统

### 内容划分

| 免费（所有用户） | 付费解锁（¥1.99 / $0.99） |
|---|---|
| 人格类型名称 + 代号 | 15 维度详细得分与文字解读（`DIM_EXPLANATIONS`） |
| 一句话简介 | 个性化发展建议 |
| 稀有度星级 | 去除本次结果页所有广告 |
| 带水印分享图 | 高清无水印分享图 |

雷达图得分可见，但每个维度的文字解读被 CSS 模糊遮罩覆盖，遮罩上方显示付费按钮。

### 支付方式

| 用户群 | 支付方式 | 说明 |
|---|---|---|
| 国内（中文环境） | 面包多 / 爱发电赞赏链接 | 零开发成本，扫码即付，个人当天可用 |
| 海外 | Stripe Checkout | 托管支付页，注册当天可用，支持全球信用卡 |

检测方式：根据 `navigator.language` 判断，`zh` 开头走国内支付，其余走 Stripe。

### 付费流程

1. 用户提交答案 → 5 秒插屏广告 → 显示基础结果（类型名 + 简介 + 星级 + 雷达图）
2. 维度文字解读区域被模糊遮罩覆盖，显示「解锁完整报告 ¥1.99 / $0.99」按钮
3. 点击按钮 → 国内弹出赞赏二维码，海外跳转 Stripe Checkout
4. 支付成功 → 移除遮罩、隐藏广告、生成无水印分享图

### 付费验证

- **Stripe**：Checkout Session 成功后跳转回 `?paid=1&session_id=xxx` 的结果页，前端调 `/api/verify` 验证 session 有效性后解锁
- **面包多/爱发电**：信任模式——用户付款后点击「我已支付」按钮即解锁。短期爆发场景，少量白嫖可接受。

### 新增 API

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/create-checkout` | POST | 接收 `{ type }` 人格类型，创建 Stripe Checkout Session，返回 `{ url }` 跳转地址 |
| `/api/verify` | POST | 接收 `{ session_id }`，调用 Stripe API 验证支付状态，返回 `{ paid: true/false }` |

需要新增环境变量：
- `STRIPE_SECRET_KEY` — Stripe 密钥
- `STRIPE_PRICE_ID` — Stripe Price 对象 ID

## 三、裂变拉新

### 分享图水印

- **免费用户**：底部加文字水印「SBTI人格测试 → sbti.jiligulu.xyz」+ 小型 QR 码
- **付费用户**：无水印，保留 QR 码（用户自愿分享，QR 码本身是拉新入口）

### 分享引导

结果页两个强引导按钮：
1. **「邀请朋友测试，看你们的相性」**——生成对比邀请链接（复用 `CompareUtil`），免费功能，目的是拉人
2. **「分享到朋友圈/Twitter」**——保存分享图到相册

### 裂变循环

```
用户A做测试 → 分享带水印图/邀请链接
  → 用户B扫码 → 做测试 → 分享...
  → 用户A付费 → 无水印图 → 更好看 → 更愿意分享
```
